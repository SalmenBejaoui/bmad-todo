Hybrid Options — Evaluation Guide

Option A — Section Labels
Most explicit. "Active · 2" and "Done · 2" labels leave no ambiguity. Slightly more UI chrome but very readable. Good for users who want clear status at a glance.
Option B — Silent Divider
Most minimal. The visual gap + rule separates the groups without labelling them. The strikethrough + muted text on completed items does the communicating. Purest expression of "silence is signal."
Option C — Collapsible Done
Done tasks hidden by default behind a toggle. Maximum focus on active tasks. Adds one interactive element (the toggle) but could be a natural fit if the done list grows long.

Evaluation Guide

Direction 1 — Clean Centre
The baseline. Warm neutral, single accent, task text primary. Closest to Things 3. Most directly aligned with the design decisions made.
Direction 2 — Floating Cards
Separates input and list into distinct cards. More visual structure. Slightly heavier chrome.
Direction 3 — Minimal Stark
Black and white only. Numbered tasks, count-first header. Maximum reduction. No colour at all.
Direction 4 — Warm Paper
Warmest palette. Tasks float on white cards inside an off-cream background. Cosy, slightly journalistic.
Direction 5 — Dark Mode
Full dark theme with the same indigo accent. For users who prefer dark interfaces.
Direction 6 — Sidebar Layout
Adds a persistent sidebar for filtering (All / Active / Done). More structured. Slight scope-creep risk.
Direction 7 — Compact Dense
Maximum density. Active and done tasks separated by a visual break. Smaller type, tighter spacing.
Direction 8 — Large Type
Personalised greeting, large headline type, generous padding. Warmest and most personal feel.
